# 多账号 / 多实例定时计划说明

本项目现在支持给每个 ECS 实例单独设置每日运行窗口，用于多账号、多实例按定时计划使用阿里云 CDT。

运行窗口是实例级配置，不限制实例数量。添加 2 台、3 台或更多服务器时，每台都会按照自己配置的 `schedule_start` 和 `schedule_end` 独立执行。

## 工作方式

安装脚本仍然会写入一个全局巡检任务：

```cron
*/5 * * * * /opt/scripts/venv/bin/python /opt/scripts/monitor.py >> /opt/scripts/monitor.log 2>&1 #aliyun_monitor
```

`monitor.py` 每 5 分钟运行一次，但每次会先读取该实例自己的计划时段：

- 当前时间在计划时段内：执行原有流量止损逻辑，流量安全则启动/保持运行，超标则关机。
- 当前时间不在计划时段内：如果实例正在运行，则自动关机；如果已经关机，则保持关机。
- 未启用计划时段的实例：保持原来的全天监控行为。

计划时间使用服务器本地时间。请先确认服务器时区，例如中国时间应为 `Asia/Shanghai`。

## 多台服务器定时计划示例

如果有 3 台服务器，可以这样分成每天 3 段：

实例 A：

```json
{
    "name": "Server-A",
    "schedule_enabled": true,
    "schedule_start": "00:00",
    "schedule_end": "08:00"
}
```

实例 B：

```json
{
    "name": "Server-B",
    "schedule_enabled": true,
    "schedule_start": "08:00",
    "schedule_end": "16:00"
}
```

实例 C：

```json
{
    "name": "Server-C",
    "schedule_enabled": true,
    "schedule_start": "16:00",
    "schedule_end": "00:00"
}
```

## 两台服务器各运行 12 小时示例

实例 A：

```json
{
    "name": "Server-A",
    "schedule_enabled": true,
    "schedule_start": "00:00",
    "schedule_end": "12:00"
}
```

实例 B：

```json
{
    "name": "Server-B",
    "schedule_enabled": true,
    "schedule_start": "12:00",
    "schedule_end": "00:00"
}
```

这样每天 `00:00-12:00` 运行 A，`12:00-00:00` 运行 B。

## 跨天窗口

支持跨天窗口，例如：

```json
{
    "schedule_enabled": true,
    "schedule_start": "20:00",
    "schedule_end": "08:00"
}
```

表示每天晚上 20:00 开始运行，第二天早上 08:00 结束。

## 修改方式

重新运行 `install.sh`，检测到已有配置后选择：

```text
1) 添加新的监控实例 (Add)
2) 删除已有监控实例 (Delete)
3) 暂停/恢复监控实例 (Pause/Resume)
4) 修改实例定时运行窗口 (Schedule)
5) 更新脚本并重置所有配置 (Update & Reset)
6) 退出脚本 (Exit)
```

也可以直接编辑 `/opt/scripts/config.json`，给对应实例增加或修改：

```json
"schedule_enabled": true,
"schedule_start": "00:00",
"schedule_end": "12:00"
```

## 注意事项

- 如果多个实例时间段重叠，它们会同时运行。
- 如果时间段之间有空档，空档期间没有实例运行。
- `schedule_start` 和 `schedule_end` 相同会被视为全天运行。
- 时间格式必须是 `HH:MM`，例如 `09:00`，不要写成 `9:00`。
- 该功能只控制实例是否处于计划运行窗口内，流量超标止损逻辑仍然优先生效。
- 日报会显示每个实例的计划时段，便于检查排班是否正确。
